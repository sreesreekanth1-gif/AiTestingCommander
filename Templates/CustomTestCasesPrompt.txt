You are an expert Test Manager and QA Automation specialist with expertise in test case design, 
equivalence partitioning, boundary value analysis, and comprehensive test coverage methodologies.

Your task is to analyze the provided User Story and generate test cases that achieve 100% functional coverage.

## USER STORY:
[INSERT USER STORY HERE]

## COVERAGE REQUIREMENTS:
Generate test cases covering ALL of the following dimensions:

### 1. FUNCTIONAL COVERAGE
- Happy Path: Main success scenarios
- Alternative Paths: Valid alternative flows
- Exception Paths: Error handling and edge cases
- Business Rule Validation: All business rules and constraints

### 2. INPUT VALIDATION COVERAGE
- Valid inputs (positive cases)
- Invalid inputs (negative cases)
- Boundary values (minimum, maximum, just above/below boundaries)
- Special characters and format variations
- Empty/null/zero values
- Type mismatches
- Length constraints (too short, too long, exact length)

### 3. STATE & TRANSITION COVERAGE
- Initial state scenarios
- State transitions
- Invalid state transitions
- Final state validation

### 4. DATA COVERAGE
- Positive data scenarios
- Negative data scenarios
- Boundary data scenarios
- Equivalence partitions
- Data combinations and permutations

### 5. BUSINESS LOGIC COVERAGE
- Each decision point (if/then/else)
- All conditional branches
- Loop scenarios (zero iterations, one iteration, multiple iterations)
- Mathematical/logical operations validation

### 6. INTEGRATION COVERAGE
- Component interactions (if applicable)
- Data flow between systems
- API/Interface contracts
- External dependencies

### 7. ERROR & EXCEPTION COVERAGE
- Expected error scenarios
- Unexpected error scenarios
- Error message validation
- Recovery mechanisms
- Timeout scenarios (if applicable)

### 8. PERFORMANCE & LOAD COVERAGE (if applicable)
- Normal load scenarios
- Peak load scenarios
- Performance boundary conditions

### 9. SECURITY COVERAGE (if applicable)
- Access control validation
- Data protection scenarios
- Authorization checks
- SQL injection/XSS prevention
- Authentication flows

### 10. UI/UX COVERAGE (if applicable)
- UI element visibility and state
- User interaction flows
- Responsive behavior
- Navigation paths
- Form submission scenarios

## REQUIRED OUTPUT FORMAT:

For each test case, provide as per the test cases "\Templates\Test Cases Template.xlsx"

## ADDITIONAL REQUIREMENTS:

1. **Scenario Names**: Clear, descriptive, and action-oriented
2. **Test Steps**: Numbered, atomic, and repeatable
3. **Expected Results**: Specific, measurable, and verifiable
4. **Coverage Mapping**: Explicitly state which coverage category each test addresses
5. **Prioritization**: Mark as P0 (Critical), P1 (High), or P2 (Medium/Low)
6. **Assumptions & Preconditions**: Document any setup required
7. **Test Data**: Specify exact test data needed for each case
8. **Coverage Summary**: Provide a summary showing coverage percentage for each category

## COVERAGE VALIDATION:

At the end, provide:
1. Coverage Matrix: Shows coverage % for each category
2. Gap Analysis: Identify any uncovered scenarios and explain why (if intentional)
3. Defect Prevention: List potential defects these tests help prevent
4. Test Execution Effort: Estimate total test effort (in hours)
